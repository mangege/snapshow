# Snapshow 友好型字幕分屏设计文档 (Jieba 版)

> 日期：2026-04-04
> 状态：待实现
> 目标：通过引入 `jieba` 分词和智能平衡算法，优化字幕分屏效果，避免出现单屏字数极少且破坏词语完整性的情况。

## 1. 核心变更 (Core Changes)

### 1.1 依赖引入
- **`requirements.txt`**：添加 `jieba>=0.42.1`。
- **懒加载策略**：在 `split_text` 内部或 `tui.py` 顶部导入 `jieba`。考虑到 TUI 的启动速度，建议在 `on_mount` 中预热分词引擎。

### 1.2 `split_text` 逻辑重写
- **基于词语的贪婪合并**：使用 `jieba.cut` 获取分词。遍历词语列表，尝试将其填入当前屏，直至达到 `max_chars`。
- **词组保护**：当一个词语（哪怕长度为 1-2）加入后会导致超过 `max_chars` 时，该词语将被完整推移至下一屏。
- **硬性兜底**：若单个词语本身长度超过 `max_chars`，则被迫在内部切断。

## 2. 智能平衡算法 (The Balanced Splitter)

### 2.1 均衡判定
- 当一段长句被分割后，若最后一屏的字数比例低于 `max_chars * 0.3`（例如：10 字限制下，末尾只剩下 1-3 个字），触发平衡检查。

### 2.2 重分配机制 (Redistribution)
- 算法将尝试从倒数第二屏末尾“借”出一个或多个词语。
- 目标：使最后两屏的字数差异尽可能缩小，使视觉展示更平稳均衡。
- 示例：`[10个字] + [2个字]` -> `[6个字] + [6个字]`（假设分词点允许）。
