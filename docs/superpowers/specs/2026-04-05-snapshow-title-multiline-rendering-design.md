# Snapshow 标题片多行自动换行设计文档

> 日期：2026-04-05
> 状态：待实现
> 目标：实现标题片内部文字根据 `max_chars` 智能自动换行，确保长标题在视频开头的视觉美感与语义完整性。

## 1. 核心逻辑 (Logic)

### 1.1 换行处理 (Auto-Wrapping)
- **触发条件**：在生成视频合成序列（FFmpeg 指令构建）时。
- **处理方式**：
    - 获取当前项目的 `title` 及 `max_chars`。
    - 调用 `snapshow.utils.split_text_smart` 算法对标题进行语义切分。
    - 将切分后的片段列表使用 **换行符 `\n`** 进行拼接。
- **示例**：
    - 输入：`"专为普通投资者打造的宽指基金筛选网站"` (max_chars=10)
    - 处理后：`"专为普通投资者打造的\n宽指基金筛选网站"`

### 1.2 FFmpeg 渲染适配 (Rendering)
- **对齐方式**：
    - **全局居中**：`x=(w-text_w)/2:y=(h-text_h)/2`。
    - **内部中对齐**：设置 `text_align=center`，确保多行文字在视觉上垂直对称。
- **字体与样式**：继承当前配置中的标题字体大小和颜色设置。

## 2. 交互与影响 (Interaction)

### 2.1 TUI 预览隔离
- **零干扰原则**：本次修改仅作用于视频合成阶段。TUI 的分段预览列表 **保持不变**（不显示标题行），以维持现有的“图片-文案”对齐心智模型。

### 2.2 配置兼容性
- 自动适配用户在 TUI 底部手动设置的 `max_chars` 数值。

## 3. 实现路径 (Implementation Path)

- **`snapshow/timeline.py`**：在 `build_timeline` 中处理标题段落的 `text` 字段。
- **`snapshow/video.py`**：确保 `drawtext` 逻辑能够正确解析并渲染带换行符的标题。

---

**预期效果**：生成的视频标题不再因为过长而溢出，而是根据语义优雅地分为多行居中显示，质感显著提升。
